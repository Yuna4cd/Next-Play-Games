# Supabase Backend

This folder contains the backend setup for task persistence with Supabase.

## Files

- `supabase/001_tasks_schema.sql`
  Creates the `tasks` and `task_comments` tables, enables RLS, and adds per-user policies.
- `supabase/002_task_comments_and_constraints.sql`
  Patch migration for projects that already ran the earlier schema version.

## Required Supabase setup

1. Create a Supabase project.
2. In the Supabase dashboard, enable Anonymous Sign-Ins.
3. Run `backend/supabase/001_tasks_schema.sql` in the SQL editor.
4. If you already ran an older version of `001_tasks_schema.sql`, also run `backend/supabase/002_task_comments_and_constraints.sql`.
5. Add frontend env vars in `Kanbanboard/.env.local`:

```env
VITE_SUPABASE_URL=https://your-project-ref.supabase.co
VITE_SUPABASE_PUBLISHABLE_KEY=your-publishable-key
```

## Notes

- RLS is enabled and every task is tied to `auth.uid()` via `user_id`.
- The app uses anonymous auth so each browser session gets its own task set.
- Task comments are stored in `task_comments`, not as a JSON blob on `tasks`.
- The app supports the default statuses `todo`, `in_progress`, `in_review`, `done`, and also supports custom column ids generated by the UI.
